# 轉盤抽籤程式 (Wheel of Names) 開發計畫

這個計畫旨在 `draw` 資料夾底下實作一個具有視覺震撼力、動畫流暢的轉盤抽籤網頁，滿足讀取 `706.txt` 學生名單、單次抽出 1 到 4 人、以及隱藏/保留抽出學生的需求。

## User Review Required

> [!IMPORTANT]
> 請確認以下設計與您的需求是否一致：
> 1. **關於一次抽出多位 (1~4 人)：** 轉盤的視覺特性一次只能停在一個區塊。當選擇抽出 2~4 人時，我將設定轉盤會**轉動一次，停在第一位中獎者的名字上**，當轉盤停下後，彈出視窗會**自動揭曉包含第一位在內的 N 位被抽出學生**。
> 2. **檔案讀取方式：** 預設會嘗試使用 `fetch` 讀取同資料夾底下的 `706.txt`。但若您是以直接雙擊 `index.html` (即 `file://` 協定) 的方式開啟，瀏覽器會因為安全性阻擋讀取。因此，我也會在畫面上提供一個**「選擇檔案」的按鈕**作為備案，讓您可以手動選取 `706.txt` 來載入名單。

## Proposed Changes

### 1. `draw/index.html` (主網頁結構)

- 建立現代化的 UI 佈局。
- 包含左側（或上方）的控制面板：
  - 檔案載入區（自動載入或手動上傳）。
  - 設定一次抽出的數量：下拉選單 (1 ~ 4)。
  - 抽籤後處理方式：切換開關 (隱藏已抽出者 vs 留在名單中)。
  - 抽出紀錄列表。
- 包含右側的大型轉盤畫布 (Canvas) 以及轉盤中央的指針。

### 2. `draw/style.css` (視覺設計與動畫)

- 套用深色背景，並將轉盤本身設計為高彩度、高對比的配色（如多彩色環），打造活潑且具現代感的抽籤氛圍。
- 實作中獎時的煙花特效 (Confetti) 與轉盤旋轉的流暢動畫體驗。

### 3. `draw/app.js` (核心抽籤邏輯)

- **名單解析**：讀取 `706.txt` 後，解析每一行的座號與姓名。
- **畫布渲染**：根據目前名單的人數（例如 26 人），將轉盤 Canvas 劃分為等份並填上不同顏色與姓名。
- **物理旋轉引擎**：使用 `requestAnimationFrame` 配合 ease-out 緩動函數，製作逼真的轉盤減速效果。
- **抽籤演算法**：
  1. 隨機選出 N 個不重複的學生。
  2. 計算第一個選出學生在轉盤上的對應角度。
  3. 啟動轉盤，強制將最終停止角度對準第一位學生。
  4. 轉盤停止後展示 N 位學生。
  5. 若選「隱藏已抽出者」，則將這些學生從名單移除，並重新繪製轉盤。
